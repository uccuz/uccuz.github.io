GCP雲端介紹

Posted on 2021-06-01 Edited on 2021-07-02

什麼是雲端

所謂雲端就是建置好軟硬體、資料或是服務，透過網路，就能夠共享軟硬體資源，使用者只需連上網路，即可享受該服務，而雲端具有以下幾個好處

  • 不需維護硬體及建置網路環境
  • 可按需購買，更好控制成本
  • 透過網路可從任何位置存取服務

雲端服務架構

雲端提供了各式各樣的服務，可以切分成三個不同模式

  • 基礎設施即服務 ( Infrastructure as a Service )
    • 提供儲存空間、記憶體、網路等基礎硬體，使用者以量計費
    • 如AWS EC2、Digital Ocean提供虛擬機服務
  • 平台即服務 ( Platform as a Service )
    • 提供執行環境或開發環境，讓使用者發展自己的應用
    • 如Cloud SQL、Heroku提供資料庫以及程式部屬
  • 軟體及服務 ( Software as a Service )
    • 將軟體提供給使用者，只需連接網路就能隨時隨地使用
    • 如Google Map、Youtube

GCP介紹

Google Cloud Platform 是 Google 所提供的雲端服務集合，預先準備好各式服務的基本元件，讓使用者能夠快速開發應用。

企業使用GCP

以蘋果公司來說，我們使用的備份服務iCloud其實背後是用Google Cloud來運作的

小型開發者使用GCP

若需要完成小型Slide Project，GCP除了可以提供網路、運算的基本資源，也提供以作好的現成服務如資料庫、AutoML等等，讓小型開發者能輕鬆建立好使用環境，快速部屬自己的應用

小結

今天講完了雲端的基本介紹，下一期開始將帶大家實際針對雲端的幾個服務進行操作

11/23-GCP實作

Overview

建立GCP專案，利用 Cloud SQL 建立雲端資料庫，並利用 App Engine 架設 Nodejs Web Server

Setup

  1. 進入 GCP 網站
  2. 登入google帳號
  3. 點擊免費試用建立帳號

建立GCP專案

  1. 點選上方選取專案 -> 新增專案

  1. 輸入專案名稱後點擊建立

建立MySQL server

  1. 點擊 console 左上角 -> 下拉找到資料庫欄位 -> 點選SQL

  1. 點選建立執行個體

  1. 選擇 MYSQL

  1. 輸入執行個體資訊
    • 執行個體ID: sqlserver
    • 根密碼: <自己設定密碼>
    • 區域: asia-east1(台灣)
    • 分區: asia-east1-a
    • 資料庫版本: MYSQL 5.7

  1. 點擊顯示設定選項

  1. 下拉找到機器類型和儲存空間點擊並更改設定
    • 機器類型點選變更 -> 選取 db-f1-micro
    • 儲存空間容量更改為10GB

  1. 點選建立

  1. 等待直到 MySQL server 建立完成 (需要幾分鐘的時間)

建立SQL資料庫並開啟API

  1. 在左側主要執行個體 -> 點選資料庫

  1. 點選建立資料庫

  1. 設定資料庫名稱為 webdb -> 點擊建立

  1. 確認webdb資料庫建立成功

  1. 點擊上方搜尋欄位 -> 輸入 Cloud SQL Admin API -> 找到 Cloud SQL Admin API 並按下確認

  1. 點擊啟用

利用Cloud Shell建立Table

  1. 進入 Cloud Shell Editor 網站

  2. 確認右上角編輯器與終端機皆已開啟

  1. 點擊終端機開始輸入指令

  2. 列出所有創建的 project -> 找到剛建立的 PROJECT_ID

  1. 設定使用此 project

  1. 連線到剛建立的 SQL Server

  1. 輸入密碼並按下確認

  1. 列出所有資料庫

  1. 切換使用資料庫為 webdb

  1. 建立 table

  1. 列出資料表欄位名稱

  1. 離開 SQL Server

建立App Engine並開啟API

  1. 回到 GCP Console 頁面
  2. 點擊 console 左上角 -> App Engine -> 資訊主頁

  1. 點擊建立應用程式

  1. Region 選擇 asia-east2 -> 點選建立應用程式

  1. Language 選擇 Node.js -> 點選繼續

  1. 點擊上方搜尋欄位 -> 輸入 App Engine Admin API -> 找到 App Engine Admin API 並按下確認

  1. 點擊啟用

部屬App Engine應用

  1. 點擊 console 左上角 -> 下拉找到資料庫欄位 -> 點選SQL

  1. 記下執行個體連線名稱

  1. 回到 Cloud Shell Editor 介面

  2. 點擊終端機輸入指令下載專案

  1. 點擊上方編輯器 DSCNCU-NodeJSWeb 資料夾 -> 開啟 app.yaml 檔案

  1. 更改 app.yaml 檔案，內容如下

    • env_variables:
      • INSTANCE_CONNECTION_NAME: <你的執行個體連線名稱>
      • DB_USER: root
      • DB_DATABASE: webdb
      • DB_PASS: <你設定的密碼>
    • beta_settings:
      • cloud_sql_instances: <你的執行個體連線名稱>
  • 修改完成後記得按下Ctrl + S 儲存

  1. 點擊終端機輸入指令

  2. 回到家目錄

  1. 變更目錄到專案位置
  1. 部署App Engine應用
  1. 輸入y並按下 enter ，即可開始部屬 (需要幾分鐘的時間)

  1. 部屬成功後，將網址複製到瀏覽器中並送出

  1. 檢查網站是否正常運作，可以嘗試新增或刪除資料

刪除專案

  1. 回到 GCP Console 頁面
  2. 選擇要刪除的專案
  3. 點擊右上角 -> 專案設定

  1. 點選關閉
  2. 輸入專案ID後點選關閉
